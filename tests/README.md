# Permission Verification Tests

## Purpose

These tests verify that configuration files generated by la-docker-compose have the correct ownership (UID/GID 1000) to be readable by containers.

## Why This Matters

All containers in **la-docker-images** are hardcoded to run as UID/GID 1000. If config files are created with different ownership, containers will fail to start with permission errors.

## Running Tests

### After config generation:

```bash
# Use default path (~/la-docker-compose)
ansible-playbook tests/playbooks/test-uid-permissions.yml

# Use custom path
ansible-playbook tests/playbooks/test-uid-permissions.yml -e compose_dir=/path/to/output
```

### Expected Output:

```
✅ All files have correct ownership
```

### If test fails:

```
❌ Permission violations detected
❌ Directory /path/config/cas has UID=1001 GID=1001 (expected 1000:1000)
```

**Fix:** Review `roles/la-compose/defaults/main.yml` and ensure `docker_container_uid` and `docker_container_gid` are set to 1000.

## Integration with CI/CD

This test should be added to the Jenkins pipeline **after** config-gen.yml and **before** deploy.yml:

```groovy
stage('Verify Permissions') {
    steps {
        sh '''
            ansible-playbook tests/playbooks/test-uid-permissions.yml \
                -e compose_dir=/data/la-docker-compose
        '''
    }
}
```

## Files Checked

- All directories under `config/`
- All files under `config/`
- Presence of critical service configs (cas, collectory, userdetails, etc.)

## Exit Codes

- **0**: All files have correct ownership
- **2**: Permission violations detected (Ansible failure)
- **4**: Config directory doesn't exist (run config-gen.yml first)
