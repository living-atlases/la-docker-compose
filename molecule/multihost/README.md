# molecule/multihost — multi-host scope-leak check

Fast (~0.1s/host), inventory-only check that catches the bug class behind the
long CI red streak (#86→#214): an inter-service dependency hostname resolving to
`localhost`/`127.0.0.1` on a deploy host because a `[<service>:vars]` override
only reaches the `.<service>` alias, **not** the `.docker_compose` host the
deploy actually runs on.

## What it does

For each `docker_compose` host in the 3-host fixture it:

1. Applies the role's own `normalize-hostnames.yml` (pure `set_fact`,
   `localhost → la_*` container aliases — the same transform the real render
   does, and what `molecule/unit` exercises).
2. Asserts that no inter-service dependency variable (DB hosts, nginx
   `*_upstream_host`, Solr/ZooKeeper/Cassandra/ES hosts) still resolves to
   `localhost`/`127.0.0.1`.

A value still pointing at loopback **after** normalization is a genuine scope
leak: in the docker-compose model every dependency is reached over the docker
network (`la_*`) or a real host — never loopback, even when co-located.

It renders nothing and deploys nothing: no docker, no host prep, no `/data` or
`/var/log` writes. (A full multi-host *render* is impractical off-host — the
service roles chown to deploy users `ubuntu`/`nginx` and build images.)

## Run

```bash
molecule test -s multihost          # via molecule (also runs in watch, Step 1b)

# or standalone:
ANSIBLE_ROLES_PATH=roles:ala-install/ansible/roles ansible-playbook \
  molecule/multihost/converge.yml \
  -i inventories/testing/lademo-multihost/lademo-inventories/lademo-inventory.ini \
  --limit docker_compose
```

The local `watch-and-test.sh` runs it as a **non-fatal** Step 1b — a leak warns
but does not block the single-host deploy.

## Fixture

`inventories/testing/lademo-multihost/lademo-inventories/lademo-inventory.ini` —
generated by `yo living-atlas` from a 3-host topology (`.yo-rc.json` in the
parent dir): `la-mh-1` (collectory/hub/biocache/cas/logger/lists/alerts/branding),
`la-mh-2` (images/spatial/geoserver/doi/data-quality), `la-mh-3`
(bie/bie-index/regions/sds/namematching/pipelines). No secrets inline (DB
passwords live in the gitignored `*-passwords.ini`, not needed by this check).

## Extend

Add a variable-name pattern to `mh_dep_key_regex` in `converge.yml` to cover a
new inter-service dependency. To refresh the topology, re-run `yo living-atlas`
against the multihost `.yo-rc.json` and re-commit the inventory.
