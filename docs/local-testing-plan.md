# Local Testing Plan: generator-living-atlas + la-docker-compose

## Goal

Enable local container-based deployments using the autogenerated inventory from
`generator-living-atlas` as the source of truth, eliminating manual duplication
of inventory variables.

## Inventories

### `inventories/local/` (CI / lint testing)

Hand-crafted inventory for `validate-local.sh` and syntax checks. Does **not**
require `generator-living-atlas`. Used by:

- `scripts/validate-local.sh`
- Molecule scenarios
- Playbook syntax checks in CI

### `inventories/testing/` (integration testing)

Generator-produced inventory for actual local deployment testing. Files:

| File | Description |
|------|-------------|
| `lademo-dev-docker-inventory.ini` | All hosts mapped to `127.0.0.1`; Docker container hostname overrides set |
| `lademo-local-extras.ini` | Extra vars / overrides for local testing |

Secrets are intentionally cleared (CAS keys, DOI passwords) — they are
auto-generated on first deploy.

## Workflow

### Prerequisites

```bash
# In the generator output directory (e.g. ~/projects/lademo/)
ls *-dev-docker-inventory.ini   # should exist after running yo living-atlas
```

### Run via ansiblew (recommended)

```bash
cd ~/projects/lademo/
./ansiblew --alainstall=../ala-install --ladocker=../la-docker-compose --docker-local --nodryrun all
```

Flags:
- `--docker-local` — selects `*-dev-docker-inventory.ini`
- `--ladocker=<dir>` — uses `la-docker-compose` playbooks instead of per-service ala-install playbooks

### Run directly

```bash
cd la-docker-compose/
ansible-playbook \
  -i ~/projects/lademo/lademo-dev-docker-inventory.ini \
  -i ~/projects/lademo/lademo-local-extras.ini \
  -i ~/projects/lademo/lademo-local-passwords.ini \
  -e deployment_type=container \
  playbooks/config-gen.yml
```

## Variable Sources

Docker container hostnames (e.g. `solr_host`, `cas_db_hostname`) come from
`[docker_compose_hosts:vars]` in the generated `*-dev-docker-inventory.ini`.
The generator template (`quick-start-inventory.ini`) includes all required
overrides:

```ini
[docker_compose_hosts:vars]
solr_host = la_solr
cassandra_host = la_cassandra
elasticsearch_host = la_elasticsearch
image_db_host = la_mysql
data_quality_db_host = la_mysql
cas_db_hostname = la_mysql
# ... etc
```

## Validation

After config generation:

```bash
# Check generated compose file
ls /tmp/la-compose-*/docker-compose/docker-compose.yml

# Validate with docker compose
docker compose -f /tmp/la-compose-*/docker-compose/docker-compose.yml config

# Start services
docker compose -f /tmp/la-compose-*/docker-compose/docker-compose.yml up -d
```

## Updating inventories/testing/

When regenerating the inventory with `yo living-atlas --replay`:

1. Copy new `*-dev-docker-inventory.ini` to `inventories/testing/`
2. Clear secrets before committing (CAS webflow keys, DOI password)
3. Copy new `*-local-extras.ini`

**Never commit real passwords or signing keys.**
